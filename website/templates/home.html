The Caeser cipher is a simple substitution cipher that shifts each letter in the plaintext by a fixed number of positions down the alphabet. It belongs to the class of monoalphabetic substitution ciphers, where each letter of the alphabet is replaced by another letter in a fixed way. In the case of the Caesar cipher, the shift value, also known as the key, is typically a single digit integer between 1 and 25. The encryption process involves replacing each letter in the plaintext with the letter that is located a fixed number of positions down the alphabet. For example, if the shift value is 3, the letter A would be replaced by D, B by E, and so on. The resulting ciphertext is then transmitted over an insecure channel to the recipient, who can then decipher it by applying the reverse transformation. The Caesar cipher is vulnerable to brute-force attacks and frequency analysis, and can be easily broken by a skilled cryptanalyst.

The Atbash cipher is a simple substitution cipher that is used to encode a message by replacing each letter of the alphabet with its reverse. For instance, A would map to Z, and B would map to Y. Here are the steps to use the Atbash cipher:
1. Write down your plaintext that you want to decrypt.
2. Write down the alphabet
3. Directly below the alphabet that you wrote, write the reverse alphabetic. This will be used for reference. It should look like this: ABCDEFGHIJKLMNOPQRSTUVWXYZ 4. For every letter in the plaintext, replace it with the corresponding letter from the reverse alphabet (below the letter in the normal alphabet).
It should be noted that the atbash cipher is one of the least safe and easiest to crack encryption methods as you need to simply reverse the alphabet again on each letter in the ciphertext.

A Monoalphabetic cipher is a simple substitution cipher that uses a fixed replacement rule to replace each letter in the plaintext with another letter. The key to the cipher is the substitution rule, which specifies which letter should be replaced with which other letter. One example of a monoalphabetic cipher is a cipher that we have seen before - the atbash cipher. This is because it maps the alphabet, to the reverse alphabet - ZYXWVUTSRQPONMLKJIHGFEDCBA. Another example is mapping the alphabet to the keys in each row of your (QWERTY) keyboard: ABCDEFGHIJKLMNOPQRSTUVWXYZ -> QWERTYUIOPASDFGHJKLZXCVBNM.

The Vigenere cipher a subsitution cipher where every letter is replaced by the sum of the positions of the original letter and the corresponding letter in the key in the alphabet. If the length of the key is shorter than the length of the message, the key is repeated in length until it reaches the length of the message. Let us look at an example: If the message is "Hello", and the key is "key", the new key will become "keyke" as the length of the key is now the same length of the message (5). Now, we should look at the position of the message and the key in the alphabet. It will look like this (ABCDEFGHIJKLMNOPQRSTUVWXYZ) (HELLO) -> 8 5 12 12 15, and the key becomes (KEYKE) -> 11 5 25 11 5. We will now add the positions of each letter to each other -> 8+11 = 19, 5+5 = 10, 25+12 = 37%26 = 11, 12+11 = 23, 15+5 = 20. It is to be noticed that when a sum of positions exceeds 26 (length of the alphabet) we will need to take that number mod 26 to achieve a "wrapping around effect" and get the position from the beginning. The final thing need to do is to subtract one from our final numbers as this is performing a shift, which means that we include the previous letter as the final part of the shift. The final numbers become 18, 9, 10, 22, 19, which translates to -> RIJVS. This is now our encrypted text!

The Trithemius cipher is a substitution ciper where each letter in the word is shifted by its position in the word.
